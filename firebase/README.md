Firebase Realtime Database rules (초안)

요약(한국어):
- 기본적으로 DB 전체는 접근 금지입니다.
- `public` 경로는 읽기 허용(예: 공개 멤버 요약 등).
- 민감한 데이터(스테이트, 토너먼트, 히스토리 등)는 인증된 관리자만 쓰기/읽기를 허용합니다.
- `admins` 경로에 관리자들의 Firebase uid를 true로 기록하고, 규칙에서 이를 검사하여 관리자 권한을 확인합니다.

권장 배포 절차:
1. Firebase Console에서 테스트 프로젝트에 먼저 rules를 적용하고 동작을 확인하세요.
2. 서비스 계정(관리자)으로 `admins/<uid>`를 하나 등록하여 브라우저에서 테스트 쓰기/읽기를 검증하세요.
3. 필요한 경우 읽기 권한을 더 개방(`ttgo_members` 읽기 허용 등)하거나, 쓰기 권한을 세분화(특정 경로만 관리자 허용)합니다.
4. 운영 환경에서는 Firebase Authentication을 설정하고, 관리자 계정 관리는 서버 측(Cloud Function 또는 별도 관리 대시보드)을 통해 제어하세요.

주의사항:
- 현재 클라이언트 앱은 전체 데이터셋을 `db.ref('ttgo_history').set(...)` 같은 방식으로 덮어쓰는 코드가 있습니다. 이 경우 쓰기를 관리자 전용으로 제한해야 데이터 삭제 위험을 줄일 수 있습니다.
- 규칙 적용 후 클라이언트 쪽 인증(로그인) 코드가 필요합니다. anonymous auth는 가능하지만, 관리자 권한 분리를 위해서는 정상 계정 기반 인증이 필요합니다.

필요하면 이 README를 바탕으로 더 세부적인 규칙(예: 사용자별 쓰기만 허용, 특정 경로에 제한된 필드만 수정 허용)을 만들어 적용해줄게.
